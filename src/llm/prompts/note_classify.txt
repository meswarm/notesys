### 核心任务
1.  **分类判定**：仔细阅读提供的笔记内容，理解其核心主题、目的和语境。
2.  **精准匹配**：从“可用分类”列表中，为笔记选择最贴切的一级分类（`category`）和二级分类（`subcategory`）。
3.  **标题生成**：根据笔记内容，提炼一个简洁、概括性强且符合命名规范的标题，作为文件名。

### 输出要求
- **格式**：必须输出严格的、无额外字符的 JSON 对象。
- **结构**：
    ```json
    {
      "category": "所选一级分类名称",
      "subcategory": "所选二级分类名称",
      "title": "生成的笔记标题"
    }
    ```

### 具体执行规则与标准
1.  **分类选择优先级**：
    *   **首要原则**：必须优先从“可用分类”列表中进行选择。`category` 和 `subcategory` 的值必须与列表中提供的名称完全一致。
    *   **匹配逻辑**：判断分类时，应综合考虑笔记的**核心主题**（如技术、生活、工作）、**内容类型**（如教程、会议记录、想法）和**具体领域**（如编程、健身、项目管理）。
    *   **无匹配处理**：仅当笔记内容与“可用分类”列表中所有选项的匹配度均低于合理阈值（例如，主题完全不符或属于全新未知领域）时，方可自行设置一级目录和二级目录

2.  **标题（`title`）生成标准**：
    *   **长度**：严格控制在 3 至 15 个字符（一个汉字计为一个字符）之间。
    *   **语言**：可使用中文或英文，确保清晰易懂。同一标题内不建议中英文混合。
    *   **禁止字符**：绝对不得包含以下任何字符：`/ \ : * ? " < > |` 以及换行符。
    *   **命名规范**：
        *   应准确概括笔记的核心内容或结论。
        *   避免使用过于宽泛的词汇（如“笔记”、“记录”）。
        *   对于教程或步骤类内容，标题应体现关键动作或目标（例如：“Python安装与环境配置指南”）。
        *   对于会议或讨论记录，标题应包含核心议题和日期（例如：“2023-10-27产品迭代方案讨论”）。
        *   对于灵感或想法，标题应点明核心观点（例如：“关于用户增长策略的新思考”）。

3.  **“无匹配”情况下的输出规范**：
    *   如果且仅如果判定为无匹配，则输出格式变更为：
        ```json
        {
          "category": "推荐的一级分类名称",
          "subcategory": "推荐的二级分类名称",
          "title": "生成的笔记标题"
        }
        ```
    *   `category` 和 `subcategory` 字段应填写你根据笔记内容逻辑推断出的、最合理的分类名称，即使它不在原列表中。
    *   同时，必须在 JSON 对象后紧跟一个清晰的文本说明，解释推荐此新分类的原因。例如：
        > **分类推荐说明**：原分类列表缺少“航空航天”领域。根据笔记内容涉及火箭发动机原理，推荐新增一级分类“科学技术”，二级分类“航空航天工程”。

{categories_text}

- **待分类笔记内容**：
{markdown_content}
